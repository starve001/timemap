## 一、CHGIS（中国历史地理信息系统）

### 基本信息

| 项目 | 详情 |
|------|------|
| **机构** | 哈佛大学费正清中国研究中心 + 复旦大学历史地理研究中心 |
| **当前版本** | CHGIS V6 |
| **时间范围** | 前221年（秦统一）— 1911年（清末） |
| **数据格式** | Shapefile (.shp)，UTF-8 / GBK 编码，WGS84 坐标系 |
| **许可** | **免费学术使用**，商业使用需另行书面协议 |

### 核心数据集

| 数据集 | 说明 | 下载链接 |
|--------|------|---------|
| **Time Series** | 时间序列数据（秦—清） | [doi:10.7910/DVN/Q9VOF5](https://dataverse.harvard.edu/dataset.xhtml?persistentId=doi:10.7910/DVN/Q9VOF5) |
| **Prefecture Points** | 州/府级治所点 | [doi:10.7910/DVN/WW1PD6](https://dataverse.harvard.edu/dataset.xhtml?persistentId=doi:10.7910/DVN/WW1PD6) |
| **Prefecture Polygons** | 州/府级政区面 | [doi:10.7910/DVN/I0Q7SM](https://dataverse.harvard.edu/dataset.xhtml?persistentId=doi:10.7910/DVN/I0Q7SM) |
| **1820 Time Slice** | 1820年全国政区 | 同 Dataverse |
| **1911 Time Slice** | 1911年全国政区 | 同 Dataverse |

### 数据文件命名规则

```
v6_time_cnty_pts_utf_wgs84.shp   → 县级行政点（时间序列）
v6_time_pref_pts_utf_wgs84.shp   → 州/府级行政点
v6_time_pref_pgn_utf_wgs84.shp   → 州/府级政区面（疆域）
```

### 获取步骤

1. 访问 [CHGIS V6 Dataverse](https://dataverse.harvard.edu/dataverse/chgis_v6)
2. 注册 Harvard Dataverse 账号（免费）
3. 下载 Time Series + Prefecture Polygons 数据集
4. 用 QGIS 打开 .shp 文件，导出为 GeoJSON
5. 用 `mapshaper` 或 `ogr2ogr` 做拓扑简化，减少体积

### ⚠️ 许可注意事项

- ✅ 学术研究、教育用途：免费使用，需注明出处
- ❌ 商业用途：需联系 CHGIS 管理委员会获取商业许可
- ❌ 中国大陆用户：需额外获得复旦大学历史地理研究中心许可

## 二、CBDB（中国历代人物传记资料库）

### 基本信息

| 项目 | 详情 |
|------|------|
| **机构** | 哈佛大学 + 中研院史语所 + 北京大学 |
| **人物数量** | **657,909人**（截至2026年5月） |
| **时间范围** | 主要为 7—19 世纪（唐—清） |
| **数据格式** | SQLite 数据库 (.db) |
| **许可** | **免费使用，无限制，学术使用** |

### 获取方式

| 方式 | 说明 |
|------|------|
| **GitHub 仓库** | [cbdb-project/cbdb_sqlite](https://github.com/cbdb-project/cbdb_sqlite) |
| **HuggingFace** | [cbdb/cbdb-sqlite](https://huggingface.co/datasets/cbdb/cbdb-sqlite) |
| **直接下载** | [latest.zip](https://huggingface.co/datasets/cbdb/cbdb-sqlite/blob/main/latest.zip) |

### 核心表结构

```
BIOG_MAIN          → 人物主表（姓名、生卒年、籍贯、朝代）
BIOG_ADDR_DATA     → 人物地址关联
ADDRESSES          → 地名标准化表（含经纬度）
ENTRY_DATA         → 入仕途径（科举、荫补等）
KINSHIP_DATA       → 亲属关系
ASSOC_DATA         → 社会关系（师友、同年等）
OFFICE_DATA        → 官职履历
POSTING_DATA       → 任职地点
STATUS_DATA        → 社会地位
TEXTS              → 相关文献
```

### 优势

- ✅ 完全免费，无使用限制
- ✅ SQLite 格式，直接可用
- ✅ 包含经纬度坐标
- ✅ 持续更新（2026年6月最新版）

## 三、替代方案

### 3.1 朝代疆域面数据

| 方案 | 数据源 | 许可 | 适用性 |
|------|--------|------|--------|
| ⭐ CHGIS Prefecture Polygons | Harvard Dataverse | 学术免费 | **最佳**，精确到州/府级 |
| 谭其骧《中国历史地图集》数字化 | 学术扫描 | 受版权保护 | 参考用，不可直接用 |
| Natural Earth | naturalearthdata.com | Public Domain | 现代边界，历史疆域不适用 |
| GADM | gadm.org | 学术免费 | 现代行政区划 |
| **手绘简化疆域** | 自行绘制 | 自主版权 | **原型阶段最佳** |

### 3.2 历史事件数据

| 方案 | 说明 |
|------|------|
| **自建结构化数据集** | 按朝代整理重大事件（时间/地点/人物/描述），JSON/YAML 格式 |
| Wikipedia API | 抓取中文维基百科历史事件条目，LLM 结构化 |
| 二十四史开放数据 | 部分学术机构提供结构化文本 |
| 维基数据 (Wikidata) | SPARQL 查询历史事件，CC0 许可 |

### 3.3 河流/地形底图

| 方案 | 数据源 | 许可 |
|------|--------|------|
| **Natural Earth** | 河流、湖泊、地形 | Public Domain ✅ |
| OpenStreetMap | 历史河流参考 | ODbL |
| 中国历史河流变迁 | 学术资料 | 参考用 |

## 四、推荐策略

```
Phase 1（原型）     → 手绘简化疆域 GeoJSON + 自建事件 JSON + CBDB 人物数据
Phase 2（MVP）      → 集成 CHGIS Shapefile → GeoJSON 管线 + CBDB 全量导入
Phase 3（完整版）   → CHGIS 全量时间序列 + 瓦片服务 + AI 辅助内容提取
```

| 都城/重要城市 | 自建 | GeoJSON Points | ✅ 已准备 |
| 主要河流 | Natural Earth | GeoJSON Lines | ✅ 已准备 |

## 五、数据管线架构

```
┌─────────────────────────────────────────────────┐
│  数据源                                          │
│  ├── CHGIS (.shp) ──→ ogr2ogr ──→ GeoJSON      │
│  ├── CBDB (.db)  ──→ Prisma  ──→ PostgreSQL     │
│  ├── Natural Earth ──→ 直接引用 GeoJSON           │
│  └── 自建事件 (.json) ──→ 直接导入               │
├─────────────────────────────────────────────────┤
│  处理层                                          │
│  ├── mapshaper（拓扑简化）                        │
│  ├── tippecanoe（生成矢量瓦片）                   │
│  └── PostGIS（空间索引）                         │
├─────────────────────────────────────────────────┤
│  服务层                                          │
│  ├── Martin（矢量瓦片服务）                       │
│  ├── Next.js API Routes（BFF聚合）               │
│  └── 静态 GeoJSON CDN                            │
└─────────────────────────────────────────────────┘
```
